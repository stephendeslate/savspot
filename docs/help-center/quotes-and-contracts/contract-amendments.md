# Contract Amendments

Sometimes contract terms need to change after a contract has been signed. SavSpot supports a structured amendment workflow that keeps a clear history of every change and requires re-signing when terms are modified.

## Modifying Existing Contracts

To amend a signed contract:

1. Navigate to `/contracts` and open the contract you need to modify.
2. Click **Amend Contract** to create an editable copy of the current version.
3. Make your changes to the contract text, terms, or pricing.
4. Review the highlighted differences between the original and amended versions.
5. Save the amendment and send it to the client for re-signing.

> **Tip:** Only amend contracts when genuinely necessary. Frequent amendments can erode client confidence. For minor clarifications, consider a private message instead.

## Amendment Workflow

The amendment process follows a clear path:

| Step | Action | Result |
|------|--------|--------|
| 1 | You initiate an amendment | A new draft version is created |
| 2 | You edit the contract terms | Changes are tracked against the previous version |
| 3 | You send the amendment | Client receives an email with the updated contract |
| 4 | Client reviews changes | Differences from the original are highlighted |
| 5 | Client signs the amendment | The new version becomes the active contract |

If the client declines the amendment, the original signed contract remains in effect.

## Version History

Every contract maintains a complete version history. You can:

- View all previous versions of a contract in chronological order.
- Compare any two versions side by side to see exactly what changed.
- Download any version as a PDF for your records.
- See who initiated each change and when.

Version history is immutable. Previous versions cannot be edited or deleted, which ensures a reliable audit trail.

## Re-signing Amended Contracts

Amended contracts require a fresh signature from the client. The re-signing process works the same as the original signing flow:

1. The client receives an email notification about the amendment.
2. They review the updated contract with changes highlighted.
3. They sign electronically to accept the new terms.
4. Both parties receive a confirmation with the updated signed contract.

> **Tip:** When sending an amendment, include a brief note explaining why the change is needed. Clients are more likely to sign promptly when they understand the reason.
