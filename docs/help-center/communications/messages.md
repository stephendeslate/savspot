# Messages

SavSpot includes a built-in messaging system that lets you communicate with your clients directly from the platform. Keep all client conversations in one place alongside their booking history and profile information.

## Accessing Messages

Navigate to **Messages** (`/messages`) from the main navigation to view all your conversations. The messages interface shows:

- A list of conversation threads on the left
- The selected conversation's message history on the right
- Client details and booking context alongside the conversation

## Conversation Threads

Each client has a single conversation thread, making it easy to find past communications. Threads are organized by most recent activity, with unread messages highlighted.

| Feature | Description |
|---|---|
| Thread list | All client conversations sorted by latest message |
| Unread indicator | Visual badge showing new messages |
| Search | Find conversations by client name or message content |
| Client context | View the client's upcoming and past bookings alongside messages |

## Sending Messages

To send a message to a client:

1. Navigate to **Messages** (`/messages`).
2. Select an existing conversation or click **New Message**.
3. If starting a new conversation, search for and select the client.
4. Type your message in the text field at the bottom.
5. Click **Send** or press Enter.

The client receives your message via their preferred notification channel (email, SMS, or in-app).

> **Tip:** Use messages for appointment-specific communication, follow-ups, or answering client questions. For recurring communications like booking confirmations and reminders, set up automated templates instead.

## Message History

All messages are stored and searchable. You can scroll through the full history of any conversation to review past communications. Messages show:

- The sender (you or the client)
- Timestamp of when the message was sent
- Read status

> **Tip:** Team members with the Staff role can only see messages for their own clients. Admins and Owners can view all message threads across the workspace.
